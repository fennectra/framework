1 Exercice 1 Beginner

Create a complete CRUD

Le framework fournit une commande CLI pour generer automatically toute la structure CRUD d'une entite. Lancez la commande suivante :

./forge make:all Article --roles=admin

This command generates the following files:

FileRole
app/Models/Article.phpThe ORM model with properties and relations
app/Controllers/ArticleController.phpThe controller with index, show, store, update, delete methods
app/Dto/Article/ArticleRequest.phpThe validation DTO for incoming requests
routes/article.phpREST routes linked to the controller

Goal: Customize the generated controller to add pagination on the index() method.

Solution 
class ArticleController
{
    public function index(): array
    {
        $page    = (int) ($_GET['page'] ?? 1);
        $perPage = (int) ($_GET['per_page'] ?? 15);
        $offset  = ($page - 1) * $perPage;

        $articles = Article::query()
            ->orderBy('created_at', 'DESC')
            ->limit($perPage)
            ->offset($offset)
            ->get();

        $total = Article::query()->count();

        return [
            'data'       => $articles,
            'page'       => $page,
            'per_page'   => $perPage,
            'total'      => $total,
            'last_page'  => (int) ceil($total / $perPage),
        ];
    }
}

2 Exercice 2 Beginner

Routes & Dynamic Params

Given the following URL pattern:

/blog/{slug}/comments/{id}

Goal: Write the route definition and the corresponding controller method signature.

Solution

Route definition:

// routes/blog.php

$router->get(
    '/blog/{slug}/comments/{id}',
    [BlogCommentController::class, 'show']
);

Controller method:

class BlogCommentController
{
    /**
     * Dynamic parameters {slug} and {id}
     * sont injectes automatically par le router.
     */
    public function show(string $slug, int $id): array
    {
        $post = Post::query()
            ->where('slug', $slug)
            ->firstOrFail();

        $comment = Comment::query()
            ->where('id', $id)
            ->where('post_id', $post->id)
            ->firstOrFail();

        return [
            'post'    => $post,
            'comment' => $comment,
        ];
    }
}
Parameter order: Dynamic parameters are injected in the order they appear in the route pattern. {slug} first, {id} second.

3 Exercice 3 Intermediate

Custom Middleware

Goal: Create a MaintenanceMiddleware that returns an HTTP 503 (Service Unavailable) response if the MAINTENANCE_MODE environment variable is set to true.

Reminder: A middleware intercepts the request before it reaches the controller. It can either let the request through or return a response directly.
Solution 
namespace App\Middleware;

class MaintenanceMiddleware
{
    /**
     * Check if the site is in maintenance mode.
     * If so, return 503 immediately.
     * Otherwise, pass the request to the next middleware.
     */
    public function handle(callable $next): mixed
    {
        if (env('MAINTENANCE_MODE') === 'true') {
            http_response_code(503);
            return [
                'error'   => 'Service Unavailable',
                'message' => 'The site is under maintenance. Please come back soon.',
            ];
        }

        // Not in maintenance: pass to the next middleware
        return $next();
    }
}

Global registration (in index.php):

$app->router()->addGlobalMiddleware(
    MaintenanceMiddleware::class
);

Or on a route group:

$router->group(['middleware' => [MaintenanceMiddleware::class]], function ($r) {
    $r->get('/api/status', [StatusController::class, 'index']);
});

4 Exercice 4 Intermediate

ORM Relations

Given the following tables:

-- categories
CREATE TABLE categories (
    id    INT PRIMARY KEY AUTO_INCREMENT,
    name  VARCHAR(100) NOT NULL
);

-- products
CREATE TABLE products (
    id            INT PRIMARY KEY AUTO_INCREMENT,
    name          VARCHAR(255) NOT NULL,
    category_id   INT NOT NULL,
    price         DECIMAL(10,2) NOT NULL,
    FOREIGN KEY (category_id) REFERENCES categories(id)
);

Goal: Write both Model classes with their relations, then a query to get all products with their category name where the price is above 100.

Solution

Category Model:

namespace App\Models;

use App\ORM\Model;
use App\ORM\Attributes\HasMany;

class Category extends Model
{
    protected static string $table = 'categories';

    public int $id;
    public string $name;

    /** A category has many products */
    #[HasMany(Product::class, 'category_id')]
    public array $products;
}

Product Model:

namespace App\Models;

use App\ORM\Model;
use App\ORM\Attributes\BelongsTo;

class Product extends Model
{
    protected static string $table = 'products';

    public int $id;
    public string $name;
    public int $category_id;
    public float $price;

    /** A product belongs to a category */
    #[BelongsTo(Category::class, 'category_id')]
    public ?Category $category;
}

Query with relation:

// Retrieve products with their category, price > 100
$products = Product::query()
    ->with('category')
    ->where('price', '>', 100)
    ->get();

// Each product has its category loaded
foreach ($products as $product) {
    echo $product->name . ' — ' . $product->category->name;
}

5 Exercice 5 Intermediate

DTO Validation

Goal: Create a RegisterRequest DTO with the following validation rules (via PHP attributes):

FieldRules
emailRequired, valid email format
passwordRequired, minimum 8 characters
nameRequired, minimum 2 characters, maximum 50
ageMinimum 18
Solution 
namespace App\Dto\Auth;

use App\Validation\Attributes\Required;
use App\Validation\Attributes\Email;
use App\Validation\Attributes\Min;
use App\Validation\Attributes\Max;

class RegisterRequest
{
    public function __construct(
        #[Required]
        #[Email]
        public readonly string $email,

        #[Required]
        #[Min(8)]
        public readonly string $password,

        #[Required]
        #[Min(2)]
        #[Max(50)]
        public readonly string $name,

        #[Min(18)]
        public readonly ?int $age = null,
    ) {}
}

Usage in the controller:

class AuthController
{
    public function register(RegisterRequest $request): array
    {
        // Le DTO est automatically valide par le framework.
        // If validation fails, a 422 is returned with errors.

        $user = User::create([
            'email'    => $request->email,
            'password' => password_hash($request->password, PASSWORD_BCRYPT),
            'name'     => $request->name,
            'age'      => $request->age,
        ]);

        return ['user' => $user, 'message' => 'Registration successful'];
    }
}

6 Exercice 6 Advanced

Event-driven architecture

Goal: When a user is created, dispatch an event. Create a listener that sends a welcome email and another that logs the creation. Show the complete flow.

Solution

1. The event:

namespace App\Events;

class UserCreated
{
    public function __construct(
        public readonly int $userId,
        public readonly string $email,
        public readonly string $name,
    ) {}
}

2. Listener — Welcome email:

namespace App\Listeners;

use App\Events\UserCreated;

class SendWelcomeEmail
{
    public function handle(UserCreated $event): void
    {
        // Send the email via a mail service
        mail(
            $event->email,
            'Welcome ' . $event->name,
            'Thank you for registering on our platform!'
        );
    }
}

3. Listener — Logging:

namespace App\Listeners;

use App\Events\UserCreated;

class LogUserCreation
{
    public function handle(UserCreated $event): void
    {
        error_log(sprintf(
            '[%s] Nouvel utilisateur cree : #%d (%s)',
            date('Y-m-d H:i:s'),
            $event->userId,
            $event->email,
        ));
    }
}

4. Listener registration (at boot):

// app/Listeners/ — automatic discovery, or:
$dispatcher = $app->container()->get(EventDispatcher::class);
$dispatcher->listen(UserCreated::class, SendWelcomeEmail::class);
$dispatcher->listen(UserCreated::class, LogUserCreation::class);

5. Dispatch from the controller:

class AuthController
{
    public function __construct(
        private EventDispatcher $events,
    ) {}

    public function register(RegisterRequest $request): array
    {
        $user = User::create([...]);

        // Dispatch the event — both listeners fire
        $this->events->dispatch(new UserCreated(
            userId: $user->id,
            email:  $user->email,
            name:   $user->name,
        ));

        return ['user' => $user];
    }
}
Worker reminder: Never call $dispatcher->listen() in a controller! Listeners are registered at boot only, otherwise they accumulate on each request.

7 Exercice 7 Advanced

Multi-database query

Objectif : Interrogez la base job pour les factures, en faisant une jointure avec les utilisateurs de la base by default. Montrez comment gerer les requetes cross-database.

Solution

Connection configuration (.env):

# Base by default
DB_DATABASE=app_main

# Secondary "job" database
DB_JOB_HOST=127.0.0.1
DB_JOB_DATABASE=app_job
DB_JOB_USERNAME=root
DB_JOB_PASSWORD=secret

Invoice Model on the "job" connection:

namespace App\Models;

use App\ORM\Model;

class Invoice extends Model
{
    protected static string $connection = 'job';
    protected static string $table      = 'invoices';

    public int $id;
    public int $user_id;
    public float $amount;
    public string $status;
}

Cross-database query with raw SQL join:

use App\ORM\Database;

// We use the "job" connection for the main query
$pdo = Database::connection('job');

$sql = '
    SELECT
        i.id          AS invoice_id,
        i.amount,
        i.status,
        u.name        AS user_name,
        u.email       AS user_email
    FROM app_job.invoices i
    INNER JOIN app_main.users u ON u.id = i.user_id
    WHERE i.status = :status
    ORDER BY i.amount DESC
';

$stmt = $pdo->prepare($sql);
$stmt->execute(['status' => 'pending']);
$results = $stmt->fetchAll();
Cross-database: MySQL allows cross-database joins if the user has permissions on both. Prefix tables with the database name: app_job.invoices, app_main.users.

8 Exercice 8 Intermediate

Complete API

Goal: Design the architecture of a mini blog API with the following constraints:

  • Models: Post, Comment, User
  • Routes: Full CRUD for posts, comments nested under posts
  • Auth: only the author of a post can edit or delete it

Architecture only, no full code.

Solution

Models and relations:

// User -> hasMany(Post), hasMany(Comment)
// Post -> belongsTo(User), hasMany(Comment)
// Comment -> belongsTo(User), belongsTo(Post)

User     { id, name, email, password }
Post     { id, user_id, title, slug, body, created_at }
Comment  { id, user_id, post_id, body, created_at }

Routes:

// ── Auth ──
POST   /api/register                          → AuthController@register
POST   /api/login                             → AuthController@login

// ── Posts (CRUD) ──
GET    /api/posts                             → PostController@index
GET    /api/posts/{id}                        → PostController@show
POST   /api/posts            [auth]           → PostController@store
PUT    /api/posts/{id}       [auth, owner]    → PostController@update
DELETE /api/posts/{id}       [auth, owner]    → PostController@delete

// ── Comments (imbriques) ──
GET    /api/posts/{postId}/comments           → CommentController@index
POST   /api/posts/{postId}/comments  [auth]   → CommentController@store
DELETE /api/comments/{id}            [auth, owner]  → CommentController@delete

Authorization middleware (owner):

// OwnerMiddleware verifies that the connected user
// is indeed the author of the resource before allowing
// modification or deletion.

class OwnerMiddleware
{
    public function handle(callable $next): mixed
    {
        $post = Post::findOrFail($id);

        if ($post->user_id !== $authUser->id) {
            throw new HttpException(403, 'Forbidden');
        }

        return $next();
    }
}
