1 Lazy Initialization in App.php Intermediate

In FrankenPHP worker mode, the application stays in memory between requests. Initializing all services at boot is wasteful: many are never used for a given request. The lazy-init pattern defers service creation to the first access.

Eager vs Lazy — Comparison
Eager (before)
class App
{
    public function boot(): void
    {
        // All services created at boot
        $this->mailer = new Mailer($config);
        $this->cache = new RedisCache();
        $this->storage = new S3Storage();
        $this->broadcaster = new Broadcaster();
        // 4 connections open even if unused
    }
}
Lazy (after)
class App
{
    public function mailer(): Mailer
    {
        // Created only on first call
        return $this->mailer
            ??= new Mailer($config);
    }

    public function cache(): Cache
    {
        return $this->cache
            ??= new RedisCache();
    }
}
??= operator (null coalescing assignment). PHP 8.3+ allows writing $this->service ??= new Service(). The service is instantiated only once, then reused for all subsequent worker requests.

2 Reflection Cache in the Router Intermediate

The Router uses PHP Reflection to read controller attributes (#[RateLimit], #[FileUpload], etc.). In worker mode, this information is constant between requests. A bounded LRU cache avoids recomputing them.

How the LRU cache works 
class Router
{
    // Cache bounded to 200 entries (LRU eviction)
    private static array $reflectionCache = [];
    private const int MAX_CACHE_SIZE = 200;

    private function getMethodAttributes(string $controller, string $method): array
    {
        $key = $controller . '::' . $method;

        if (isset(self::$reflectionCache[$key])) {
            // Cache hit: move to front (LRU)
            $value = self::$reflectionCache[$key];
            unset(self::$reflectionCache[$key]);
            self::$reflectionCache[$key] = $value;
            return $value;
        }

        // Cache miss: Reflection + storage
        $ref = new ReflectionMethod($controller, $method);
        $attrs = $ref->getAttributes();

        // LRU eviction if cache is full
        if (count(self::$reflectionCache) >= self::MAX_CACHE_SIZE) {
            array_shift(self::$reflectionCache);
        }

        self::$reflectionCache[$key] = $attrs;
        return $attrs;
    }
}
Mandatory bound in worker mode. Without a limit, the cache would grow indefinitely. LRU with MAX_CACHE_SIZE guarantees constant memory consumption. After the initial warm-up, the hit rate reaches ~99%.

3 CTE PostgreSQL Advanced

Common Table Expressions (CTE) combine multiple queries into one, reducing network round-trips with the database. The framework uses them for dashboards and statistics.

Example: NF525 statistics
Without CTE (3 queries)
$invoices = DB::query(
    'SELECT COUNT(*) FROM invoices'
);
$closings = DB::query(
    'SELECT COUNT(*) FROM nf525_closings'
);
$total = DB::query(
    'SELECT SUM(total_ht) FROM invoices'
);
With CTE (1 query)
$stats = DB::query('
  WITH inv AS (
    SELECT COUNT(*) as cnt,
           SUM(total_ht) as total
    FROM invoices
  ),
  cls AS (
    SELECT COUNT(*) as cnt
    FROM nf525_closings
  )
  SELECT inv.cnt, inv.total,
         cls.cnt as closings
  FROM inv, cls
');
Network round-trips. Each SQL query involves a network round-trip (~0.5ms locally, ~2-5ms in cloud). Combining 3 queries into 1 CTE saves 2 round-trips, i.e., 1-10ms per call.

4 Optimized Middleware Pipeline Intermediate

Les middlewares stateless (sans etat entre les requetes) peuvent etre caches pour eviter de les reinstancier a chaque requete. Le pipeline detecte automatically les middlewares reutilisables.

Stateless middleware caching 
class MiddlewarePipeline
{
    private static array $instances = [];

    private function resolve(string $class): object
    {
        // Reuse stateless instances between requests
        return self::$instances[$class]
            ??= new $class();
    }
}

Middlewares caches automatically :

5 RouteCache — Static Loading Beginner

The cache:routes command pre-compiles all routes into a serialized PHP file. In worker mode, this file is loaded once at boot and reused for all requests.

Usage 
# Pre-compile the route cache
./forge cache:routes

# Result: var/cache/routes.php
# Contains all serialized routes (compiled regex, middleware, handlers)
Without cache:
  Requete ──► Load app/Routes/*.php ──► Compile regex ──► Dispatch
                    ~3ms per request

With cache:
  Boot worker ──► Load routes.php (once only)
  Requete     ──► Direct dispatch
                    ~0.1ms per request
En mode worker, ces optimisations s'accumulent sur des milliers de requetes. Un worker qui traite 10 000 requetes economise 30 secondes de compilation de routes et des centaines de milliers d'appels Reflection. Le gain reel in production est significatif.
