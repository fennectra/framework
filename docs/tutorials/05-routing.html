Routing & Middleware

The framework HTTP layer: routes, middleware pipeline, request/response & auto-generated API docs

Tutorials / 05 — Routing

1 Defining Routes Beginner

The Router exposes four HTTP-verb methods: get(), post(), put(), and delete(). Each takes a path, a controller/action pair, and optional per-route middleware.

Route files live in app/Routes/ and are auto-loaded at boot time. Dynamic segments use the {param} syntax — the value is injected into the controller method parameter with the matching name.

$router->get('/users', [UserController::class, 'index']);
$router->post('/users', [UserController::class, 'store']);
$router->put('/users/{id}', [UserController::class, 'update']);
$router->delete('/users/{id}', [UserController::class, 'delete']);
Dynamic parameters — {id} in the path becomes the regex group (?P<id>[^/]+). The router matches the named group and passes it to the controller method whose parameter is named $id.

2 Route Groups Beginner

Groups let you share a prefix, middleware, and description across several routes. Groups can be nested — prefixes and middleware accumulate, and the previous state is restored when the group callback ends.

$router->group([
    'prefix'      => '/admin',
    'middleware'  => [[Auth::class, ['admin']]],
    'description' => 'Administration',
], function ($router) {
    $router->get('/users', [AdminController::class, 'listUsers']);
});

The route above resolves to GET /admin/users with the Auth middleware requiring the admin role.

How nesting works internally

Before entering the callback, the router saves the current groupPrefix, groupMiddleware, and groupDescription. Inside the callback the new values are merged. Once the callback finishes, the previous values are restored — this makes nested groups safe.

3 Middleware Intermediate

Middleware intercept the request before (and after) it reaches the controller. Every middleware implements MiddlewareInterface with a single handle(Request, callable $next) method.

class RateLimitMiddleware implements MiddlewareInterface
{
    public function handle(Request $request, callable $next): mixed
    {
        // Check rate limit before proceeding...
        if ($this->isRateLimited($request)) {
            throw new HttpException(429, 'Too Many Requests');
        }

        return $next($request);
    }
}

Pipeline execution

The MiddlewarePipeline wraps middleware in an onion-style chain. Global middleware run first, then per-route middleware, and finally the controller action as the core handler.

Request │ ▼ ┌─────────────────────────────────┐ │ Global Middleware (CORS, etc.) │ │ ┌───────────────────────────┐ │ │ │ Route Middleware (Auth) │ │ │ │ ┌─────────────────────┐ │ │ │ │ │ Controller::action │ │ │ │ │ └─────────────────────┘ │ │ │ └───────────────────────────┘ │ └─────────────────────────────────┘ │ ▼ Response

Global vs per-route middleware

TypeRegistrationRuns on
Global $router->addGlobalMiddleware(CorsMiddleware::class) Every request
Per-route Third argument of get()/post()/etc., or via group() Only matching routes 
// Global middleware — registered once at boot
$router->addGlobalMiddleware(CorsMiddleware::class);

// Per-route middleware — only on this specific route
$router->post('/payments', [PaymentController::class, 'charge'], [
    [Auth::class, ['admin']],
    [RateLimitMiddleware::class],
]);

4 Request Object Beginner

The Request class wraps the incoming HTTP data. It is created by the router and passed through the middleware pipeline.

MethodReturnsDescription
getMethod()stringHTTP verb: GET, POST, PUT, DELETE
getUri()stringRequest path (e.g. /users/42)
getBody()?arrayParsed JSON body from php://input
getQuery($key, $default)mixedQuery string params ($_GET). Pass null for all.
getHeader($name)?stringHTTP header value (e.g. 'Authorization')
getServer($key, $default)mixedAccess $_SERVER values
withAttribute($key, $val)RequestReturns a clone with the attribute set
getAttribute($key, $default)mixedRetrieve a custom attribute
Immutable attributes — withAttribute() returns a clone of the Request. The original is never mutated. This is essential for passing data between middleware safely (e.g. the authenticated user). 
// Inside a middleware
$request = $request->withAttribute('auth_user', $user);
return $next($request);

// Inside a controller (via $_REQUEST or attribute)
$token = $request->getHeader('Authorization');
$page  = $request->getQuery('page', 1);

5 Response Handling Beginner

Controller methods simply return a value. The router handles serialization automatically:

Return typeBehaviour
arraySerialized to JSON via Response::json()
objectPublic properties extracted, then JSON
nullNo response body sent (headers only)
 
// Returning an array → auto JSON response
public function index(): array
{
    return ['users' => User::all()];
}

// Throwing HttpException → structured error response
public function show(string $id): array
{
    $user = User::find($id);
    if (!$user) {
        throw new HttpException(404, 'User not found');
    }
    return $user;
}
Never use exit() or die() to end a response. Throw an HttpException instead — the framework catches it, sets the correct status code, and keeps the worker alive.

6 OpenAPI Auto-Docs Intermediate

The framework automatically generates an OpenAPI specification from your routes and PHP 8 attributes. No manual YAML required.

Documenting your endpoints

use Framework\Attributes\ApiDescription;
use Framework\Attributes\ApiStatus;

#[ApiDescription(
    'List all users',
    'Returns a paginated list of active users.'
)]
#[ApiStatus(200, 'Successful response')]
#[ApiStatus(401, 'Unauthorized')]
public function index(): array
{
    return User::all();
}
AttributeTargetPurpose
#[ApiDescription(summary, description)]MethodEndpoint summary & detailed description
#[ApiStatus(code, description)]Method (repeatable)Document possible HTTP response codes

Accessing the docs

Two built-in endpoints are registered by the DocsController:

URLDescription
/docsInteractive Scalar UI for browsing and testing the API
/docs/openapiRaw OpenAPI JSON specification
Scalar UI is loaded from CDN and reads the /docs/openapi endpoint. Route descriptions, DTO schemas, and status codes are all generated automatically from your code.

✎ Exercise 1 Beginner

Create a route group for /api/v2 with auth middleware.

Define a group that prefixes all routes with /api/v2, applies the Auth middleware (requiring the user role), and contains at least two routes.

Solution 
use App\Middleware\Auth;
use App\Controllers\ProductController;
use App\Controllers\OrderController;

$router->group([
    'prefix'      => '/api/v2',
    'middleware'  => [[Auth::class, ['user']]],
    'description' => 'API v2',
], function ($router) {
    $router->get('/products', [ProductController::class, 'index']);
    $router->get('/products/{id}', [ProductController::class, 'show']);
    $router->post('/orders', [OrderController::class, 'store']);
});

This produces routes GET /api/v2/products, GET /api/v2/products/{id}, and POST /api/v2/orders — all protected by Auth middleware with the user role.

✎ Exercise 2 Intermediate

Create a custom LoggingMiddleware that logs request duration.

Implement MiddlewareInterface. Record microtime(true) before calling $next, then compute the elapsed time and log it with the HTTP method and URI.

Solution 
namespace App\Middleware;

use Framework\Core\MiddlewareInterface;
use Framework\Core\Request;

class LoggingMiddleware implements MiddlewareInterface
{
    public function handle(Request $request, callable $next): mixed
    {
        $start = microtime(true);

        $result = $next($request);

        $duration = round((microtime(true) - $start) * 1000, 2);

        error_log(sprintf(
            '[%s] %s %s — %sms',
            date('Y-m-d H:i:s'),
            $request->getMethod(),
            $request->getUri(),
            $duration
        ));

        return $result;
    }
}

Register it globally with $router->addGlobalMiddleware(LoggingMiddleware::class) or attach it to specific route groups.
