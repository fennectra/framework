Lifecycle & Golden Rules

Understanding the FrankenPHP worker lifecycle

Tutorials / 01 — Lifecycle

1 PHP-FPM vs Worker Beginner

In classic mode, PHP starts a new process for each request. In worker mode, the application lives indefinitely in memory.

PHP-FPM (classic) FrankenPHP Worker Request 1: Boot (once only): Boot app ──► Handle ──► Destroy Boot app ──► Load routes │ Request 2: Loop: ▼ Boot app ──► Handle ──► Destroy ┌──► beforeRequest() │ frankenphp_handle_request() Request 3: │ └── handler() ──► dispatch HTTP Boot app ──► Handle ──► Destroy │ afterRequest() │ cleanup (DB, auth, GC) Each request = fresh world └──── loop ────────────────────┘ Same app, same memory, same objects

3 Our worker, line by line Beginner

index.php — Automatic mode detection 
$app = new App();                         // 1. Created ONCE
$app->router()->addGlobalMiddleware(...);   // 2. Middleware loaded ONCE
$app->loadRoutes(...);                     // 3. Routes loaded ONCE

if (function_exists('frankenphp_handle_request')) {
    $app->runWorker();  // 4. Infinite loop
} else {
    $app->run();        // Classic mode
}
App::runWorker() — The loop with guaranteed cleanup 
$stats = new WorkerStats();                // Created once, lives forever

for ($i = 0; ...; ++$i) {
    $stats->beforeRequest();                // Memory snapshot

    \frankenphp_handle_request($handler); // Wait for a request

    $stats->afterRequest();                 // Metrics

    try {
        $dbManager->flush();                // Rollback + close connections
        Database::clearInstances();         // Clear singleton cache
    } finally {
        unset($_REQUEST['__auth_user']);   // ALWAYS executed
        \gc_collect_cycles();               // Force GC
    }
}
Why try/finally? If flush() throws an exception, the finally block guarantees that auth is cleaned up and GC runs. Without it, a user could leak into the next request.

4 The 5 Golden Rules Beginner

1. Never use exit() or die() CRITICAL

exit() kills the entire worker. All subsequent requests will fail.

Forbidden
if (!$user) {
    http_response_code(401);
    echo 'Unauthorized';
    exit;  // CRASHES the worker!
}
Correct
if (!$user) {
    throw new HttpException(
        401, 'Unauthorized'
    );
}
2. No request-scoped static state CRITICAL

Request data stored in a static property will leak into the next request.

Dangerous
class CurrentUser
{
    private static ?array $user = null;

    public static function set($u)
    {
        self::$user = $u;
        // Leaks into the next request!
    }
}
Correct
// Via $_REQUEST (cleaned by the worker)
$_REQUEST['__auth_user'] = $user;

// Or via the Request object (immutable)
$request->withAttribute(
    'auth_user', $user
);
3. Clean up DB connections IMPORTANT

An uncommitted transaction locks tables for subsequent requests. Our cleanup performs flush() automatically.

4. Event listeners = boot only IMPORTANT

Never call Event::listen() inside a controller. Listeners accumulate with each request.

Request 1: 1 listener. Request 100: 100 identical listeners. Listeners should be in app/Listeners/, discovered at boot.
5. Headers before the body IMPORTANT

FrankenPHP sends the response at the end of the handler. Set headers before any echo. Our Router already handles this correctly.
