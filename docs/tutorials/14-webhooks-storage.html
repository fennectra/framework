1 WebhookManager Intermediate

The webhook system allows sending HTTP notifications to external systems when events occur in your application. Each webhook is signed with HMAC-SHA256 to guarantee authenticity.

Automatic dispatch via Events

Les webhooks sont dispatches automatically quand un evenement est emis. The WebhookManager filters registered webhooks listening for that event:

use Fennec\Core\Webhook\WebhookManager;

// Manual webhook dispatch
WebhookManager::dispatch('order.shipped', [
    'order_id' => 42,
    'tracking' => 'FR123456789',
]);

// Ou automatically via le systeme d'events :
Event::dispatch('order.shipped', $data);
// → The WebhookManager detects registered webhooks
//   pour 'order.shipped' et les dispatch automatically
HMAC-SHA256 Signature

Each webhook request includes a signature in the X-Webhook-Signature header. The recipient can verify authenticity:

// Server-side (sending):
$payload = json_encode($data);
$signature = hash_hmac('sha256', $payload, $webhook->secret);

// Sent headers:
// X-Webhook-Signature: sha256=a1b2c3d4...
// X-Webhook-Event: order.shipped
// X-Webhook-Delivery: uuid-v4
// Content-Type: application/json

// Recipient-side (verification):
$expected = hash_hmac('sha256', $body, $secret);
if (!hash_equals('sha256=' . $expected, $signatureHeader)) {
    return response(401, 'Invalid signature');
}
Retry with exponential backoff

Le WebhookDeliveryJob retente automatically les livraisons echouees avec un backoff exponentiel :

AttemptDelayCumulative time
1Immediate0s
210s10s
330s40s
490s~2min
5270s~7min
6810s~20min
 
class WebhookDeliveryJob extends AbstractJob
{
    public int $maxRetries = 5;

    public function retryDelay(int $attempt): int
    {
        // Exponential backoff: 10 * 3^attempt
        return 10 * (3 ** $attempt);
    }
}
webhook_deliveries table. Each attempt is tracked in the webhook_deliveries table with the HTTP response status, response time and body in case of error.

2 Storage Multi-Driver Beginner

The Storage facade offers a unified API to store, read and delete files, regardless of the underlying driver: local, Amazon S3 or Google Cloud Storage.

Basic API 
use Fennec\Core\Storage\Storage;

// Store a file
Storage::put('avatars/user-42.jpg', $fileContent);

// Read a file
$content = Storage::get('avatars/user-42.jpg');

// Delete a file
Storage::delete('avatars/user-42.jpg');

// Get the public URL
$url = Storage::url('avatars/user-42.jpg');
// Local : /storage/avatars/user-42.jpg
// S3 :    https://bucket.s3.amazonaws.com/avatars/user-42.jpg
Driver configuration 
# .env — Driver local (by default)
STORAGE_DRIVER=local

# .env — Amazon S3
STORAGE_DRIVER=s3
AWS_BUCKET=my-app-uploads
AWS_REGION=eu-west-3
AWS_ACCESS_KEY_ID=AKIA...
AWS_SECRET_ACCESS_KEY=secret...

# .env — Google Cloud Storage
STORAGE_DRIVER=gcs
GCS_BUCKET=my-app-uploads
GCS_KEY_FILE=/path/to/service-account.json
Public symlink. With the local driver, run ./forge storage:link to create the public/storage → storage/ symlink and make files accessible via HTTP.

3 ImageTransformer Intermediate

The ImageTransformer offers a chainable pipeline to resize, crop and convert images. Based on the PHP GD extension, it requires no external dependencies.

Chainable pipeline 
use Fennec\Core\Image\ImageTransformer;

// Simple resize
ImageTransformer::resize('photo.jpg', 800);

// Full pipeline
ImageTransformer::make('photo.jpg')
    ->resize(800)             // Max width 800px
    ->crop(400, 400)         // Square crop
    ->format('webp')           // WebP conversion
    ->apply();                 // Apply transformations
Attribut #[FileUpload]

The #[FileUpload] attribute on a controller method enables automatic parsing of multipart/form-data requests and OpenAPI documentation:

use Fennec\Attributes\FileUpload;
use Fennec\Attributes\ImageTransform;

class AvatarController
{
    #[FileUpload]
    #[ImageTransform(width: 200, height: 200, format: 'webp')]
    public function upload(array $request): array
    {
        // $request['file'] contains the uploaded file
        $path = Storage::put(
            'avatars/' . $request['user_id'] . '.webp',
            $request['file']['content']
        );

        return ['url' => Storage::url($path)];
    }
}
Extension GD requise. Le ImageTransformer necessite l'extension PHP GD. Le Dockerfile du framework l'inclut automatically. Verifiez avec php -m | grep gd.
