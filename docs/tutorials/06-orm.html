1 Defining a Model Beginner

Each model extends Framework\Core\Model and declares its table via the #[Table] or the static property $table. Relations are simple methods that call belongsTo, hasMany ou hasOne.

User.php — Full example 
namespace App\Models;

use Framework\Attributes\Table;
use Framework\Core\Model;
use Framework\Core\Collection;

// The #[Table] attribute links the model to the SQL table
#[Table('users')]
class User extends Model
{
    // Alternative: static property (takes priority over the attribute)
    protected static string $table = 'users';

    // Automatic column casting
    protected static array $casts = [
        'is_active'      => 'bool',
        'role_id'        => 'int',
        'credit_base'    => 'json',
        'current_credit' => 'int',
    ];

    // BelongsTo relation: User -> Role
    public function role(): ?Role
    {
        return $this->belongsTo(Role::class, 'role_id');
    }

    // HasMany relation: User -> Posts
    public function posts(): Collection
    {
        return $this->hasMany(Post::class, 'user_id');
    }
}
Available static properties
PropertyTypeDefaultDescription
$tablestring''SQL table name (otherwise inferred from class name)
$primaryKeystring'id'Primary key
$connectionstring'default'DB connection name
$timestampsbooltrueAutomatic management of created_at / updated_at
$softDeletesboolfalseEnables soft delete via deleted_at
$castsarray[]Casting : int, float, bool, string, json, array, datetime
Naming convention: Si $table est vide et qu'il n'y a pas d'attribut #[Table], the framework infers the name : UserProfile → user_profiles (snake_case + pluriel).

2 CRUD Operations Beginner

Read Lire — findmdash; find, findOrFail, all, where 
// Find by primary key (returns null if not found)
$user = User::find(42);

// Find or throw an HttpException 404
$user = User::findOrFail(42);

// Retrieve all records (returns a Collection)
$users = User::all();

// Count records
$total = User::count();

// Conditional query (returns a ModelQueryBuilder)
$admins = User::where('is_active', true)
    ->where('role_id', 1)
    ->get();
Create Creer — createmdash; create, save (insert) 
// Method 1: create() — instantiates + persists in one line
$user = User::create([
    'email'     => 'alice@example.com',
    'role_id'   => 2,
    'is_active' => true,
]);
// $user->id is now populated (lastInsertId)

// Method 2: new + fill + save
$user = new User();
$user->email = 'bob@example.com';
$user->role_id = 3;
$user->save();  // INSERT — timestamps auto-filled
Event: After an insert, l'evenement App\Models\User.created is dispatched automatically.
Update Modifier — savemdash; save (update), update 
// Update an existing model
$user = User::findOrFail(42);
$user->email = 'newemail@example.com';
$user->save();  // UPDATE — only changed fields (dirty) are sent

// Check if a field has changed
$user->isDirty('email');   // true/false
$user->getDirty();           // ['email' => 'newemail@...']

// Bulk update via QueryBuilder
User::where('is_active', false)
    ->update(['is_active' => true]);
Event: After an update, l'evenement App\Models\User.updated est dispatche.
Delete Supprimer — deletemdash; delete, forceDelete, restore (soft delete) 
// Standard deletion (or soft delete if $softDeletes = true)
$user->delete();

// Permanent deletion (ignores soft delete)
$user->forceDelete();

// Restore a soft-deleted model
$user->restore();

// Check if a model is soft-deleted
$user->isDeleted();  // true/false

To enable soft delete on a model :

class Article extends Model
{
    protected static bool $softDeletes = true;
    // deleted_at is automatically excluded from SELECTs
}

Queries with deleted records :

// Include soft-deleted
$all = Article::withTrashed()->get();

// Only soft-deleted
$trashed = Article::onlyTrashed()->get();

3 Query Builder — Fluent API Intermediate

Le QueryBuilder builds SQL queries via a chainable interface. Each call returns $this, enabling fluent composition. Parameters are automatically prepared (:qb_p0, :qb_p1, ...) to prevent SQL injection.

Methodes de filtrage (WHERE) 
// where(column, value) — egalite implicite
User::where('is_active', true)

// where(column, operator, value) — operateur explicite
User::where('age', '>', 18)
User::where('name', 'LIKE', '%Alice%')

// orWhere — condition OR
User::where('role_id', 1)
    ->orWhere('role_id', 2)

// whereIn — liste de valeurs
User::where('is_active', true)
    ->whereIn('role_id', [1, 2, 3])

// whereNull / whereNotNull
User::where('is_active', true)
    ->whereNull('deleted_at')
    ->whereNotNull('email')
Tri, pagination et selection 
// orderBy(column, direction) — defaut ASC
User::where('is_active', true)
    ->orderBy('created_at', 'DESC')

// limit et offset
User::where('is_active', true)
    ->orderBy('created_at', 'DESC')
    ->limit(10)
    ->offset(20)

// select — choisir les colonnes
User::where('is_active', true)
    ->select('id', 'email', 'created_at')

// join / leftJoin
User::where('is_active', true)
    ->join('roles', 'users.role_id', '=', 'roles.id')
    ->leftJoin('profiles', 'users.id', '=', 'profiles.user_id')
Methodes terminales
MethodeRetourDescription
->get()Collection (Model) / array (QB)Toutes les lignes correspondantes
->first()?Model / ?arrayPremiere ligne (LIMIT 1 auto)
->firstOrFail()ModelPremiere ligne ou HttpException 404
->count()intCOUNT(*)
->exists()boolAu moins une ligne existe ?
->insert(array)string|falseINSERT et retourne le lastInsertId
->update(array)intNombre de lignes affectees
->delete()intNombre de lignes supprimees
->paginate(perPage, page)arrayPagination avec data + meta

Exemple de pagination :

// Pagination native — retourne data + meta
$result = User::paginate(perPage: 15, page: 2);

// $result = [
//     'data' => [ [...], [...], ... ],
//     'meta' => [
//         'total'        => 150,
//         'per_page'     => 15,
//         'current_page' => 2,
//         'last_page'    => 10,
//     ]
// ]

4 Relations Intermediate

Le framework supporte trois types de relations. Chaque methode de relation est definie dans le model et execute une requete a l'appel.

Tableau des relations
MethodeSignatureRetourExemple
belongsTo belongsTo(Related::class, ?foreignKey, ?ownerKey) ?Model Un Post appartient a un User
hasMany hasMany(Related::class, ?foreignKey, ?localKey) Collection Un User a plusieurs Posts
hasOne hasOne(Related::class, ?foreignKey, ?localKey) ?Model Un User a un Profile
Convention des cles : Si foreignKey n'est pas fourni, le framework le devine : User → user_id, UserProfile → user_profile_id.
Exemples complets 
// ─── belongsTo : Post -> User ───────────────
class Post extends Model
{
    public function author(): ?Model
    {
        return $this->belongsTo(User::class, 'user_id');
    }
}

// ─── hasMany : User -> Posts ─────────────────
class User extends Model
{
    public function posts(): Collection
    {
        return $this->hasMany(Post::class, 'user_id');
    }
}

// ─── hasOne : User -> Profile ────────────────
class User extends Model
{
    public function profile(): ?Model
    {
        return $this->hasOne(Profile::class, 'user_id');
    }
}

// ─── Utilisation ─────────────────────────────
$user = User::findOrFail(1);

$role    = $user->role();       // ?Role (belongsTo)
$posts   = $user->posts();      // Collection<Post> (hasMany)
$profile = $user->profile();    // ?Profile (hasOne)

5 Facade DB Beginner

La classe DB fournit un acces statique au DatabaseManager pour les requetes brutes et les transactions.

API de la facade DB 
use Framework\Core\DB;

// ─── QueryBuilder via DB::table() ────────────
$rows = DB::table('users')
    ->where('is_active', true)
    ->orderBy('created_at', 'DESC')
    ->limit(10)
    ->get();  // Retourne array (pas des Models)

// ─── Requete SQL brute ───────────────────────
$stmt = DB::raw(
    'SELECT u.*, r.role_name
     FROM users u
     JOIN roles r ON r.id = u.role_id
     WHERE u.email = :email',
    ['email' => 'alice@example.com']
);
$user = $stmt->fetch();  // PDO fetch associatif

// ─── Transaction ─────────────────────────────
DB::transaction(function () {
    $user = User::create(['email' => 'new@example.com', 'role_id' => 1]);

    DB::table('audit_logs')->insert([
        'action'  => 'user_created',
        'user_id' => $user->id,
    ]);
});
// Commit auto si pas d'exception, rollback sinon

// ─── Connexion specifique ────────────────────
$jobs = DB::table('jobs', 'job')->get();
$stmt = DB::raw('SELECT * FROM jobs', [], 'job');
DB::transaction($callback, 'job');
Methodes disponibles
MethodeSignatureDescription
DB::table()table(string $table, string $connection = 'default'): QueryBuilderCree un QueryBuilder pour la table
DB::raw()raw(string $sql, array $params = [], string $connection = 'default'): PDOStatementExecute du SQL brut avec parametres prepares
DB::transaction()transaction(callable $cb, string $connection = 'default'): mixedExecute dans une transaction (commit/rollback auto)
DB::connection()connection(string $name = 'default'): DatabaseAcces direct a l'objet Database

6 Multi-database Intermediate

Le framework supporte plusieurs connexions PostgreSQL simultanees. Chaque connexion est nommee et configuree via le .env.

Configuration .env 
# Connexion par defaut (nom : "default")
# Prefixe .env : POSTGRES_*
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DB=futurandco
POSTGRES_USER=myuser
POSTGRES_PASSWORD=secret

# Connexion "job" (nom : "job")
# Prefixe .env : POSTGRES_JOB_*
POSTGRES_JOB_HOST=10.0.0.50
POSTGRES_JOB_PORT=5432
POSTGRES_JOB_DB=presta15
POSTGRES_JOB_USER=user
POSTGRES_JOB_PASSWORD=pass

# Connexion "test" (nom : "test")
# Prefixe .env : POSTGRES_TEST_*
POSTGRES_TEST_HOST=localhost
POSTGRES_TEST_PORT=5432
POSTGRES_TEST_DB=test
POSTGRES_TEST_USER=myuser
POSTGRES_TEST_PASSWORD=secret
Convention : Le nom de connexion "job" correspond au prefixe POSTGRES_JOB_* dans le .env. Le mapping est automatique : POSTGRES_ + strtoupper(name) + _HOST, etc.
Utilisation dans le code 
// ─── Via DB facade ───────────────────────────
$jobs = DB::table('jobs', 'job')->get();

// ─── Via l'attribut #[Table] sur un Model ────
#[Table('jobs', connection: 'job')]
class Job extends Model
{
    // Toutes les requetes de ce model iront
    // sur la connexion "job" automatiquement
}

// ─── Ou via la propriete statique ────────────
class Job extends Model
{
    protected static string $connection = 'job';
    protected static string $table = 'jobs';
}

// Les deux fonctionnent de la meme facon :
$job = Job::find(1);       // Requete sur la DB "job"
$all = Job::all();          // Requete sur la DB "job"

7 DTOs & Validation Intermediate

Les DTOs (Data Transfer Objects) sont des classes readonly dont les parametres du constructeur portent des attributs de validation. Le Validator inspecte ces attributs via la Reflection API.

Definir un DTO avec validation 
namespace App\Dto\User;

use Framework\Attributes\Required;
use Framework\Attributes\Email;
use Framework\Attributes\MinLength;
use Framework\Attributes\MaxLength;
use Framework\Attributes\Min;
use Framework\Attributes\Max;
use Framework\Attributes\Regex;

readonly class UserStoreRequest
{
    public function __construct(
        #[Required]
        #[Email]
        #[MaxLength(255)]
        public string $email = '',

        #[Required]
        #[Min(1)]
        public int $role_id = 0,

        public bool $is_active = true,

        public ?bool $is_consent = null,
    ) {}
}
Attributs de validation disponibles
AttributParametresDescription
#[Required]?string $messageChamp obligatoire
#[Email]?string $messageAdresse email valide (filter_var)
#[MinLength(n)]int $min, ?string $messageLongueur minimale (string)
#[MaxLength(n)]int $max, ?string $messageLongueur maximale (string)
#[Min(n)]int|float $min, ?string $messageValeur minimale (numerique)
#[Max(n)]int|float $max, ?string $messageValeur maximale (numerique)
#[Regex(pattern)]string $pattern, string $messageValidation par expression reguliere
Utiliser Validator::validate() 
use Framework\Core\Validator;
use App\Dto\UserStoreRequest;

// Donnees brutes (ex: JSON decode du body)
$data = json_decode(file_get_contents('php://input'), true);

// Validation — retourne un tableau d'erreurs (vide si valide)
$errors = Validator::validate(UserStoreRequest::class, $data);

if (!empty($errors)) {
    // Retourner les erreurs 422
    throw new HttpException(422, json_encode($errors));
}

// Hydrater le DTO
$dto = new UserStoreRequest(...$data);

// Utiliser les donnees validees
$user = User::create([
    'email'     => $dto->email,
    'role_id'   => $dto->role_id,
    'is_active' => $dto->is_active,
]);
Verification de type : Le Validator verifie aussi le type de base (string, int, float, bool, array) avant d'appliquer les attributs de validation. Un int envoye comme string sera rejete.

✎ Exercices

Exercise 1 — Model Product avec soft delete Beginner

Objectif : Creer un model Product avec soft delete et une relation belongsTo vers Category.

Consignes :

  • Table : products
  • Soft delete active
  • Casts : price en float, is_published en bool, metadata en json
  • Relation : un produit appartient a une categorie via category_id
Solution 
namespace App\Models;

use Framework\Attributes\Table;
use Framework\Core\Model;

#[Table('products')]
class Product extends Model
{
    protected static bool $softDeletes = true;

    protected static array $casts = [
        'price'        => 'float',
        'is_published' => 'bool',
        'metadata'     => 'json',
    ];

    public function category(): ?Model
    {
        return $this->belongsTo(Category::class, 'category_id');
    }
}
Exercise 2 — Requete complexe avec le Query Builder Intermediate

Objectif : Ecrire une requete qui recupere tous les utilisateurs actifs avec le role 'admin', tries par date de creation decroissante, limites a 10.

Consignes :

  • Utiliser le QueryBuilder avec join
  • Filtrer sur is_active = true et role_name = 'admin'
  • Trier par created_at DESC
  • Limiter a 10 resultats
Solution 
use Framework\Core\DB;

// Solution 1 : Via DB facade (retourne des arrays)
$admins = DB::table('users')
    ->select('users.*', 'roles.role_name')
    ->join('roles', 'users.role_id', '=', 'roles.id')
    ->where('users.is_active', true)
    ->where('roles.role_name', 'admin')
    ->orderBy('users.created_at', 'DESC')
    ->limit(10)
    ->get();

// Solution 2 : Via le Model (returns a Collection de User)
$admins = User::where('is_active', true)
    ->join('roles', 'users.role_id', '=', 'roles.id')
    ->where('roles.role_name', 'admin')
    ->orderBy('users.created_at', 'DESC')
    ->limit(10)
    ->get();

// Solution 3 : Via DB::raw pour une requete SQL directe
$stmt = DB::raw(
    'SELECT u.*, r.role_name
     FROM users u
     JOIN roles r ON r.id = u.role_id
     WHERE u.is_active = :active
       AND r.role_name = :role
     ORDER BY u.created_at DESC
     LIMIT 10',
    ['active' => true, 'role' => 'admin']
);
$admins = $stmt->fetchAll();
